# 需求：设置页面集成麦克风检测功能

## 问题背景

当前机器上没有麦克风时，程序不会报错，而是录制到空白音频，SenseVoice 对空白音频会输出一个奇怪的韩语字符（实际上是语言标签噪声）。这个问题在 Linux 系统上尤为突出，因为 Linux 服务器/桌面通常没有内置麦克风。用户需要去外部网站才能检测麦克风是否正常工作，体验很差。

## 需求目标

在 whisper-input 的**设置页面**中集成一个简单的麦克风检测模块，帮助用户在不离开应用的情况下：

1. **检测麦克风是否可用**：列出当前系统可用的麦克风设备，明确提示「未检测到麦克风」或「检测到 N 个麦克风设备」。
2. **录音回放（可选但优先级高）**：让用户录制几秒钟的音频，然后在浏览器中直接播放，自行判断麦克风音质是否正常。

## 技术约束

- 设置页面是基于 `settings_server.py` 的内嵌 HTTP 服务 + 浏览器 Web UI，前端用原生 JS + HTML（无框架）。
- 后端用 Python（`sounddevice` 已是依赖，可直接用于设备列举和录音）。
- 录音回放需要把 WAV/PCM 数据传到浏览器，浏览器用 Web Audio API 或 `<audio>` 标签播放。
- 跨平台兼容：Linux（ALSA/PulseAudio/PipeWire）和 macOS。

## 验收标准

1. 设置页面出现「麦克风检测」区域。
2. 点击「检测」能列出可用麦克风设备名称；无设备时明确提示。
3. 点击「录音 & 回放」能录制 3 秒音频，完成后在页面内播放，用户可自行判断音质。
4. 不引入新的运行时依赖（`sounddevice` 已有）。
